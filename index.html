Selected Topic: Colosseum

The Colosseum is also called the Flavian Amphitheatre. It is one of the most famous monuments in Rome and one of the best known symbols of ancient Roman culture.

According to Wikipedia, the Colosseum is an elliptical amphitheatre in the centre of Rome, near the Roman Forum. It is described as the largest ancient amphitheatre ever built and the largest standing amphitheatre in the world.

Construction began under Emperor Vespasian around AD 70-72 and was completed under Titus in AD 80. Later changes were made under Domitian. The monument is connected with the Flavian dynasty, which explains the name "Flavian Amphitheatre".

Wikipedia also describes the main materials as travertine limestone, tuff and brick-faced Roman concrete. The amphitheatre could hold tens of thousands of spectators and was used for gladiatorial contests, animal hunts, public spectacles, executions, re-enactments of battles and mythological dramas.

In the medieval period the building was no longer used as an entertainment arena. It was later reused for housing, workshops, religious purposes, as a fortress, as a quarry and as a Christian shrine. These historical details are useful because they show what kind of knowledge can enrich the RDF description of the monument.

Project 1: Enriching a Knowledge Graph

The goal of Project 1 is to explore a knowledge graph with SPARQL, find a missing or weak description, and propose new RDF triples that could enrich the graph.

The ArCo search identified resources connected with "Colosseo", including this site resource:

https://w3id.org/arco/resource/Site/7982fa371295ec533db6c4dce3fc879a

This resource is useful because it is labelled "Anfiteatro Flavio-Colosseo", but its RDF description is quite small. It mainly has a type, a label/name and an address. It does not include many facts about the monument itself.

Why this gap matters

The monument has rich historical, architectural and heritage information, but the selected ArCo site resource contains only a minimal description. Adding verified triples makes the resource more useful for search, linking and comparison with other cultural heritage data.

Gap foundThe ArCo site resource exists, but the description is minimal.
Useful enrichmentAlternative names, external links, materials, coordinates, heritage status and provenance.
VerificationFacts were checked with ArCo, Wikidata, Wikipedia, UNESCO and the official Colosseum website.

SPARQL Queries

The following ArCo queries were used to find the Colosseum resource and understand what information is missing. Together they include all required SPARQL keywords.

Q1. Search Colosseum resources

PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

SELECT DISTINCT ?resource ?label WHERE {
  ?resource rdfs:label ?label .
  ?label bif:contains "'Colosseo'" .
  FILTER(REGEX(STR(?label), "Colosseo", "i"))
}
ORDER BY ?label
LIMIT 10

Example result: the query returns several Colosseum-related records, including the site resource labelled "Anfiteatro Flavio-Colosseo".

Q2. Inspect the selected site

PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

SELECT DISTINCT ?property ?value ?valueLabel WHERE {
  <https://w3id.org/arco/resource/Site/7982fa371295ec533db6c4dce3fc879a> ?property ?value .
  OPTIONAL { ?value rdfs:label ?valueLabel . }
}
ORDER BY ?property
LIMIT 20

Example result: the resource has a type, a label/name and an address. This confirms that the description is present, but not very rich.

Q3. Compare two names of the monument

PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

SELECT DISTINCT ?resource ?label WHERE {
  {
    ?resource rdfs:label ?label .
    ?label bif:contains "'Colosseo'" .
  }
  UNION
  {
    ?resource rdfs:label ?label .
    ?label bif:contains "'Anfiteatro Flavio'" .
  }
  FILTER(REGEX(STR(?label), "Colosseo|Anfiteatro Flavio", "i"))
}
ORDER BY ?label
LIMIT 15

Example result: many returned records are photographs, titles or subject labels. This helped identify the gap: ArCo mentions the Colosseum often, but the selected site resource still needs a richer description.

RDF Triples Proposed

The full RDF proposal is stored in enriched_colosseum.ttl. The main idea is to enrich the existing ArCo site resource instead of creating a totally separate record.

<https://w3id.org/arco/resource/Site/7982fa371295ec533db6c4dce3fc879a>
  skos:altLabel "Flavian Amphitheatre"@en, "Anfiteatro Flavio"@it ;
  owl:sameAs <http://www.wikidata.org/entity/Q10285> ;
  a-loc:hasSiteType ex:RomanAmphitheatre ;
  a-dd:hasMaterial ex:RomanTravertine, ex:Tuff, ex:RomanConcrete ;
  schema:geo ex:ColosseumGeometry ;
  schema:isPartOf <https://whc.unesco.org/en/list/91/> ;
  prov:wasDerivedFrom <https://dati.cultura.gov.it/sparql> .

A small RDFS extension, ex:hasHeritageRecognition, is also proposed to express heritage recognition with provenance.

Project 2: Source and Narrative Analysis

Project 2 uses knowledge graphs and LLMs to compare how the same topic is described by different sources. This part keeps the same topic, the Colosseum, and compares the way different sources frame it.

The comparison is based on sources that can be checked online: Wikipedia, the official Colosseum park page, UNESCO, ArCo and Wikidata.

Narrative focus

The same monument can be presented as an architectural achievement, a place of ancient spectacles, a protected heritage site or a modern tourist destination. Project 2 compares these frames.

Research questions

RQ1Is the Colosseum described mainly as architecture, entertainment, world heritage or tourist attraction?
RQ2Which actors are mentioned: emperors, spectators, tourists, institutions or conservation authorities?
RQ3Does adding KG triples help make LLM answers more factual?

Prompting Techniques and LLM Comparison

The same topic was tested with three prompting techniques. The prompts were written so that the model had to stay close to the evidence from ArCo, Wikidata and the textual sources.

TechniquePromptExample result
Zero-shotDescribe how the Colosseum is represented in ArCo and suggest possible RDF triples for enrichment.The model suggested useful facts, but some property names needed checking.
Few-shotExample: if a monument has a material, model it with a-dd:hasMaterial. Now suggest triples for the Colosseum.The model followed the example and produced triples closer to the chosen vocabulary.
Grounded / RAGUse only the ArCo query results, the RDF triples and the source excerpts. Mark each claim as supported by KG, text, or both.This gave the most reliable answer because unsupported claims were easier to notice.

Comparison of two language models

ModelObserved behaviourEvaluation
ChatGPTProduced clearer explanations and connected the RDF triples with the narrative comparison.Good for writing and interpretation, but still needed fact-checking.
GeminiGave shorter answers and was useful for summarising the difference between sources.Good for concise summaries, but less detailed in RDF vocabulary choices.

Evaluation and Conclusion

The KG-only approach is very factual, but it does not explain the narrative very deeply. The text-only LLM approach gives a better explanation, but it can add claims that are not directly supported. The hybrid approach is the best for this work because RDF triples keep the answer grounded, while the LLM helps compare the sources.

Final conclusion: the Colosseum is present in ArCo, but the selected site resource can be enriched with more useful RDF triples. For narrative analysis, the same verified triples can help make LLM answers more reliable.
